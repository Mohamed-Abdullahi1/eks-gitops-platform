# EKS GitOps Platform

A fully automated Kubernetes platform built on AWS using Terraform, EKS, Helm, and ArgoCD.

This project demonstrates how modern DevOps teams provision infrastructure, deploy applications, and manage environments through GitOps workflows.

The cluster, networking, ingress, DNS, TLS certificates, and application deployment are all automated through infrastructure as code.

# Infrastructure

All infrastructure is created using Terraform.

Resources provisioned include:

- VPC and networking
- Public and private subnets
- Internet gateway
- Security groups
- EKS Kubernetes cluster
- Managed node group
- IAM roles and policies
- Route53 DNS integration

Terraform allows the entire environment to be recreated from code.

---

# Kubernetes Platform

Once the cluster is created, several services are installed using Helm.

### Ingress Controller
Handles external traffic into the cluster.

### External DNS
Automatically creates DNS records in Route53 for Kubernetes ingress resources.

### Cert Manager
Automatically provisions TLS certificates using Let's Encrypt.

### ArgoCD
Implements GitOps by synchronising Kubernetes manifests from a Git repository.

---

# GitOps Workflow

ArgoCD continuously watches the Git repository and deploys any changes automatically.

Deployment flow:

1. Application manifests are pushed to GitHub
2. ArgoCD detects the change
3. ArgoCD applies the manifests to Kubernetes
4. Kubernetes updates the running application

This ensures the cluster state always matches the Git repository.

# How to Deploy

Initialise Terraform:

```bash
terraform init
```

Review infrastructure plan:

```bash
terraform plan
```

Create the infrastructure:

```bash
terraform apply
```

After the cluster is created, apply the issuer and ArgoCD application:

```bash
kubectl apply -f cert-man/issuer.yaml
kubectl apply -f argo-cd/apps-argo.yaml
```

ArgoCD will then synchronise applications automatically.
